# 阶段三：研究升级路径

研究是一个不断迭代深化的过程，从表层走向深度。

---

## 3.1 从v1.0到v3.0的演进

```
v1.0 卖方模板版
├── 财务指标罗列
├── 行业概况描述
├── 估值倍数对比
└── 问题：停在表层，无判断力

v2.0 工程国家周期版（四轴框架）
├── 工程能力深度拆解
├── 国家平台定位
├── 周期与结构判断
└── 问题：框架完整但偏静态

v3.0 情绪交易版（可交易框架）
├── 叙事强度监测
├── 资金行为分析
├── 板块轮动定位
└── 价值：可落地的交易策略
```

---

## 3.2 关键变量识别

对于每家被研究公司，需回答：

### 正向变量（什么会让它变好？）

```
正向变量（什么会让它变好？）
├── 行业资本开支上行
├── 国产设备在关键节点突破
├── 国企改革带来激励释放
└── 新业务（如：云业务）盈亏平衡
```

### 负向变量（什么会让它变差？）

```
负向变量（什么会让它变差？）
├── 下游投资长期深度下行
├── 国家工程预算系统性收缩
├── 竞争加剧导致毛利率持续下滑
└── 应收账款回收风险
```

---

## 3.3 研究深度检查清单

### v1.0 → v2.0 升级检查

- [ ] 是否完成了产业链拆解？
- [ ] 是否分析了公司制度性位置？
- [ ] 是否识别了周期属性？
- [ ] 是否评估了护城河结构？

### v2.0 → v3.0 升级检查

- [ ] 是否识别了当前市场叙事？
- [ ] 是否分析了资金行为？
- [ ] 是否定位了板块角色？
- [ ] 是否形成了可交易策略？

---

## 3.4 持续迭代节奏

```
研究是动态过程，需要定期更新：

季度更新：财务数据、机构持仓、股东户数
半年更新：行业竞争格局、毛利率趋势
年度更新：战略定位、国企改革进度、新业务拐点
事件驱动：重大订单、政策变化、技术突破
```
