# 阶段五：数据查询与验证体系

系统化的数据查询方法和关键信号识别。

---

## 5.1 WarrenQ/聚源核心查询语句模板

### 分部财务数据（识别亏损来源）

```
[股票代码] 分部财务数据 [起始年份]-[结束年份]
→ 目的：看云业务/新业务的EBIT趋势
```

### 毛利率对比（判断行业vs个体问题）

```
[代码A] [代码B] 销售毛利率(TTM) [时间范围]
→ 目的：判断是行业通病(Beta)还是个体问题(Alpha)
```

### 机构持仓变动

```
[代码] 机构持仓比例 基金持股家数 [季度]
→ 目的：观察"国家队"资金动向
```

### 股东户数变化（筹码集中度）

```
[代码] 股东户数变化 连续[N]个季度
→ 目的：判断机构是否在底部吸筹
```

### 龙虎榜活跃度

```
[代码] 龙虎榜上榜次数 近[N]个月
→ 目的：判断游资参与度
```

### 融资余额

```
[代码] 融资余额 融资余额占流通市值比 近[N]个月
→ 目的：判断杠杆资金态度
```

## 5.2 关键反转信号清单

### 适用对象

周期底部+国家平台+新业务培育类型公司

### 财务信号

```
├── 云业务/新业务EBIT连续两个季度显著收窄
├── 单季度营收环比增长>10%
├── 毛利率环比回升且不再下滑
└── 经营性现金流回正
```

### 资金信号

```
├── 股东户数锐减（筹码集中）
├── ETF逆势增持
├── 龙虎榜出现机构席位净买入
└── 融资余额止跌回升
```

### 事件信号

```
├── 新业务明确盈亏平衡时间表
├── 半导体/行业CapEx拐点上修
└── 国企改革/资产整合落地
```
